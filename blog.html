To know var, let, and const click here

var: Hoisted, not block-scoped, and function-scoped with the ability to redeclare. Not advised in contemporary JavaScript.

let:Hoisted, block-scoped, and not allowed to be redeclared inside the same block. preferred option for declaring variables in most situations.

const:Hoisted, block-scoped, and unassignable after declaration. Use for quantities or constants that shouldn't alter

Click to know about Use Cases of null and undefined

Undefined:It is used when a variable has been declared but not assigned a value or when trying to access a nonexistent object property.

Null:It is used to represent an intentional absence of any object value or as a placeholder for nonexistent or unknown objects.

Click to know about Rest API

It is a type of architectural design used to create networked applications. It makes use of HTTP methods like GET (for data retrieval), POST (for new data creation), PUT (for updated data), and DELETE (for data removal). Endpoints, which are used to identify resources in REST APIs, are URLs. Data is typically transferred in the JSON format. Secure access to these APIs is provided via authentication techniques like API keys, tokens, or OAuth. The results of API requests are indicated by HTTP status codes like 200 (for success) and 404 (for not found). Processing these requests is aided by middleware, which is frequently used in server frameworks like Express.js. To limit the number of client requests within a given time window, rate limiting is used.Additionally, APIs may incorporate version numbers in their URLs to maintain backward compatibility as they evolve.
